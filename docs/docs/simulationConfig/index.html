Simulation Settings

Simulation parameterizations and settings are accessed in the Simulation menu by clicking the Simulation Settings button icon.

Simulation Settings Edit Formâ

The image below shows the simulation settings edit form:

This form is subdivided into four distinct contexts:

  • General: where general and common information to various simulations is entered;
  • Power Flow: where the numerical solution method is selected, as well as its parameters;
  • Stability: containing time simulation options and parameters of the integration method;
  • ZIP Load: contains general options of ZIP loads in the stability study.

Base Powerâ

Base power of the system used for conversion of real data into p.u.p.u.p.u. and vice versa. Can be entered in VA, kVA, or MVA.

Tip

The base power of each element can be different from the system base power. To do this, just check the option "Use nominal power [of the element] as base," present in their respective data edit forms.

System Frequencyâ

Defines the nominal frequency of the system.

Warning!

When changing the nominal frequency, pay attention to the "Open Circuit Frequency" field of synchronous machines.

Continuous Calculationâ

Enables or disables continuous calculation for short circuit, short circuit level, and harmonic distortions calculations. Continuous calculation for power flow is always enabled.

Tip

To enable continuous calculation, press the Enable Solution button present in the Simulation menu.

With this option, the selected static calculations are automatically performed when modifying any network parameters, such as electrical data and breaker operation (removal or insertion).

Solution Methodâ

Defines the numerical solution method for the power flow study. Currently, the PSP-UFU includes the following methods:

  • Gauss-Seidel
  • Newton-Raphson
  • Hybrid Gauss-Newton

Max Iterationsâ

Maximum number of iterations a method can reach. If this value is reached, the power flow calculation is stopped with an error.

Reference Bus Angleâ

The value of the angle of the reference bus, in degrees.

Newton Inertiaâ

Value that multiplies the power corrections. Values greater than 1.01.01.0 increase the correction, and values less than 1.01.01.0 decrease the correction.

Information

Values less than 1.01.01.0 may help convergence, but increase the number of iterations and consequently the processing time of the method.

This option is enabled only for the Newton-Raphson method.

Gauss Toleranceâ

Defines the tolerance of the Gauss-Seidel for the hybrid method. This option is enabled only for the Hybrid Gauss-Newton method.

Since Newton-Raphson is considerably sensitive to initial conditions, the first iterations are calculated using Gauss-Seidel until the error is less than the tolerance specified in the âGauss Toleranceâ field. From that point, the calculation is done using the Newton-Raphson method until convergence is achieved with an error smaller than the tolerance stipulated in the âToleranceâ field.

Information

This option helps convergence of systems that cannot be solved using conventional Newton-Raphson.

Integration Stepâ

Defines the integration step for the Implicit Trapezoidal integration method.

Caution!

Very high values may generate simulation errors, while very small values will significantly increase the processing time of the stability calculation.

Simulation Timeâ

Defines the total simulation time, in seconds.

Toleranceâ

Tolerance of the iterative process (in p.u.p.u.p.u.) of the Implicit Trapezoidal integration method.

Max Iterationsâ

Maximum number of iterations of the iterative process of the Implicit Trapezoidal integration method. If the number of iterations exceeds this value, the stability calculation will be stopped with an error.

Use general composition for all system loadsâ

Enables the composition of the ZIP load for all loads that do not have individual parameterization.

Load Composition (active and reactive powers)â

Defines the composition, in percentage, of the Impedance, Current, and Constant Power components of the load. This composition can be implemented individually for active and reactive power.

Caution!

The sum of the ZIP load components for a power must be 100%, otherwise an error message will be displayed.

Information
